# KEYOB — News Section (Next.js + TypeScript + Tailwind)

A reusable, SEO/AEO-optimised News section that extends the existing KEYOB brand
(cool navy / blue-grey / cyan palette anchored on the site `theme-color` `#F5F8FA`,
with Fraunces + Space Grotesk + JetBrains Mono type).

## Routes
- `/news` — listing page. H1: *News & Perspectives on AI Operations*. Featured card,
  category filter, editorial feed with AEO answer blocks interleaved, newsletter block, final CTA.
- `/news/[slug]` — article page. Breadcrumb, article hero, semantic body (H2/H3),
  Key Takeaway box, Related Solutions, 3 Related Articles, "Discuss this with KEYOB" CTA.
- `/news/rss.xml` — RSS feed (crawlable, static).
- `/sitemap.xml` — generated by `app/sitemap.ts`, includes `/news` and every article.

## Structure
```
app/
  globals.css            Tailwind layers + brand tokens + button components
  sitemap.ts             news + all articles
  news/
    page.tsx             /news (CollectionPage + Breadcrumb + Organization schema)
    [slug]/page.tsx      /news/[slug] (Article|NewsArticle + Breadcrumb + Organization schema)
    rss.xml/route.ts     RSS feed
components/               13 reusable components (see brief)
data/articles.ts         12 sample articles + category list
lib/
  types.ts               NewsArticle data model
  seo.ts                 metadata + JSON-LD helpers (Article vs NewsArticle)
  format.ts              date formatting
tailwind.config.ts       brand palette + fonts
```

## Schema strategy
- `kind: "evergreen"` → **Article** schema (all 12 samples are evergreen guides).
- `kind: "news"` → **NewsArticle** schema (use only for genuinely timely items).
- Every article also emits **BreadcrumbList** + **Organization**.
- `/news` emits **CollectionPage** + **BreadcrumbList** + **Organization**.
- AEO: question-first answer blocks + Key Takeaway boxes use Question/Answer microdata
  and extraction-friendly phrasing.

## To run inside the real site
1. Copy `app/news`, `components`, `lib`, `data` into your Next.js (App Router) project.
2. Merge `tailwind.config.ts` tokens and the `globals.css` `@layer components` button classes.
3. Ensure the `@/` path alias points at the project root (tsconfig `paths`).
4. Replace placeholder image paths in `data/articles.ts` (`/news/*.webp`) with real
   business photography (teams at dashboards, warehouses, service teams). Until then the
   `CategoryVisual` SVG renders an on-brand geometric placeholder — no robots/brains/stock.
5. Add the News link to the primary nav and `/news/rss.xml` to `<head>` as an alternate.

## Notes
- Outcomes in sample content are qualitative/directional, not invented client metrics.
- "AI assistant" content is kept generic — no unverifiable product names.
- Replace `SITE` values in `lib/seo.ts` if the canonical host differs.
